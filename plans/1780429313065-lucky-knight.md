# Plan: `sancho` CLI — Multi-Provider LLM Delegation Tool (Go Edition)

## Goal
A CLI tool for delegating coding tasks to LLMs via multiple providers (OpenRouter, AWS Bedrock, OpenAI, Anthropic). Provides `sancho ask` for bulk reading and `sancho write` for code generation, with clean, minimal output modes.

---

## 0) Configuration — layered precedence

Resolution order (highest wins):
1. **CLI flags** (e.g. `--model`, `--api-key`)
2. **Config file** (JSON/JSONC)
3. **Environment variables**
4. **Built-in defaults**

### Environment variables

| Variable | Purpose | Default |
|---|---|---|
| `SANCHO_API_KEY` | API key (provider-specific) | — |
| `SANCHO_BASE_URL` | Provider endpoint | `https://openrouter.ai/api/v1` |
| `SANCHO_MODEL` | Model identifier | deepseek/deepseek-chat |
| `SANCHO_PROVIDER` | Provider: "openrouter", "bedrock", "openai", "anthropic" | openrouter |
| `AWS_ACCESS_KEY_ID` | AWS Bedrock access key (for bedrock provider) | — |
| `AWS_SECRET_ACCESS_KEY` | AWS Bedrock secret (for bedrock provider) | — |
| `AWS_REGION` | AWS region (for bedrock provider) | us-east-1 |

Backwards-compat fallbacks: `WORKER_API_KEY`, `WORKER_BASE_URL`, `WORKER_MODEL`, `WORKER_PROVIDER`.

### Config file

CLI searches for a config in (first match wins):
- `./.sancho.json` (project-local, favourite)
- `$HOME/.config/sancho/config.json` (user-level)

**Schema**:

```jsonc
{
  "api_key": "sk-or-...",
  "base_url": "https://openrouter.ai/api/v1",
  "model": "deepseek/deepseek-chat",
  "provider": "openrouter", // or "bedrock", "openai", "anthropic"

  "openrouter": {
    "api_key": "sk-or-...", // optional: override for this provider
    "http_referer": "https://sancho-cli.dev" // optional
  },
  "bedrock": {
    "region": "us-east-1",
    "model_mappings": {
      "claude-3-opus": "anthropic.claude-3-opus-20240307-v1:0",
      "gpt-4": "anthropic.claude-3-sonnet-20240229-v1:0"
    }
  },
  "openai": {
    "api_key": "sk-...", // optional: override for this provider
    "organization": "org-..." // optional
  },
  "anthropic": {
    "api_key": "sk-ant-..." // optional: override for this provider
  },

  "ask": { "max_tokens": 8192 },
  "write": { "max_tokens": 16384 },
  "retry": {
    "max_attempts": 3,
    "backoff_max_seconds": 5,
    "retry_status_codes": [429, 502, 503, 504]
  },
  "timeout_seconds": 120
}
```

**JSONC handling**: strip `// ...` and `/* ... */` comments with a small regex pass, then `json.Unmarshal`. No external JSONC library — keeps `sancho` dependency-free.

> **Caveat**: regex-based comment stripping is naive and will break on `//` inside string literals. For config files this is acceptable (users are expected to use sane JSONC). If stricter parsing is needed later, swap for a small hand-written tokenizer.

## 1) Commands

### `sancho ask`
Delegate bulk reading to a cheap LLM. Returns **MinLang** (token-minimal) output.

**Flags**
- `-p / --paths <files...>`: files to ingest (supports globs)
- `-q / --question <string>`: extraction/analysis query
- `-n / --max-tokens <int>`: default 8192
- `-m / --model <string>`: override model
- `--provider <string>`: override provider (openrouter, bedrock, openai, anthropic)
- `--api-key <string>`: override API key
- `--base-url <string>`: override provider endpoint
- `--json`: emit only the answer to stdout, suppress cost footer

**Behavior**
- Reads all `--paths` files (with basic glob expansion via `filepath.Glob` + walk for `**`).
- Wraps each file in `<file path="...">...</file>`.
- System prompt instructs the model to emit **MinLang**:
  - `@path:line` for locations
  - `-` for bullets
  - `=>` for key-value mappings
  - No markdown, no prose, under 120 tokens
- Prints answer to stdout; cost footer to stderr.

---

### `sancho write`
Generate code/docs from a spec, optionally style-matched to a reference file.

**Flags**
- `-s / --spec <string>`: what to write (required)
- `-c / --context <path>`: reference file for style (optional)
- `-t / --target <path>`: output file path (required)
- `-n / --max-tokens <int>`: default 16384
- `-m / --model <string>`: override model
- `--provider <string>`: override provider (openrouter, bedrock, openai, anthropic)
- `--api-key <string>`: override API key
- `--base-url <string>`: override provider endpoint

**Behavior**
- If `--context` provided, prepend it wrapped in `<reference>...</reference>`.
- System prompt: *"Generate clean, idiomatic code matching the style of any reference provided. No explanations, no markdown fences — output ONLY the file contents."*
- Strips accidental markdown fences (`` ``` ``) before writing.
- Writes to `--target`, prints `Wrote <path> (<n> chars)` to stdout (human-readable, plain text).
- Cost footer to stderr.

## 4) Architecture: file-by-file contract

Every file below has a single responsibility. What it **does** and what it **does not** do are both specified.

### `main.go`
- **Does**: declares `package main`, calls `cmd.Execute()`, returns exit code.
- **Does not**: contain business logic, flags, or HTTP calls.

### `cmd/root.go`
- **Does**: builds root `cobra.Command`, registers `ask` and `write` subcommands, wires shared config (flags, resolution).
- **Does not**: call the OpenRouter API, read files from disk, or parse JSONL.

### `cmd/ask.go`
- **Does**: bind `ask` subcommand flags (`-p`, `-q`, `-n`, `-m`, `--provider`, `--api-key`, `--base-url`, `--json`), resolve settings, expand globs, read file contents, get provider via `client.NewProvider`, call `provider.ChatCompletion`, print MinLang answer to stdout and cost footer to stderr.
- **Does not**: mutate files, perform retries directly, or implement `write` behaviour.

### `cmd/write.go`
- **Does**: bind `write` subcommand flags (`-s`, `-c`, `-t`, `-n`, `-m`, `--provider`, `--api-key`, `--base-url`), resolve settings, read optional context file, get provider via `client.NewProvider`, call `provider.ChatCompletion`, strip markdown fences, write target file, print human-readable status line to stdout and cost footer to stderr.
- **Does not**: read source files for analysis, or emit MinLang.

### `internal/client/provider.go`
- **Does**: define `Provider` interface with `ChatCompletion` method, `ChatRequest`/`ChatResponse` types, and `NewProvider(settings)` factory function that returns the appropriate provider implementation.
- **Does not**: implement provider logic, perform HTTP calls, or know about specific API formats.

### `internal/client/openrouter.go`
- **Does**: implement `Provider` interface for OpenRouter. Uses API key from config, standard Bearer auth, required headers (`HTTP-Referer`, `X-Title`), POSTs to `/chat/completions`.
- **Does not**: resolve settings, or handle AWS/OpenAI-specific auth.

### `internal/client/bedrock.go`
- **Does**: implement `Provider` interface for AWS Bedrock. Uses AWS SDK v2 credential chain, SigV4 signing, maps model names to Bedrock format.
- **Does not**: handle OpenRouter/OpenAI auth, or make unauthenticated requests.

### `internal/client/openai.go`
- **Does**: implement `Provider` interface for OpenAI. Uses standard OpenAI API format with Bearer auth.
- **Does not**: know about Bedrock or OpenRouter-specific headers.

### `internal/client/anthropic.go`
- **Does**: implement `Provider` interface for Anthropic Claude. Uses `/v1/messages` endpoint, different request/response format than chat completions.
- **Does not**: know about other providers' formats.

### `internal/config/env.go`
- **Does**: read `SANCHO_*` env vars including `SANCHO_PROVIDER`, fall back to `WORKER_*` aliases. For Bedrock, also checks `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION`. Returns a partial `Config` with zero values for unset fields.
- **Does not**: read files from disk, merge configs, or know CLI flags.

### `internal/config/file.go`
- **Does**: probe `.sancho.json` then `$HOME/.config/sancho/config.json`, strip JSONC comments with regex, unmarshal into `Config`, return zero-valued struct when file missing.
- **Does not**: merge with env or CLI, or make network calls.

### `internal/config/resolve.go`
- **Does**: merge `Config` from file, env, and CLI flags into final `Settings`; CLI flags win, then file, then env, then defaults. Detects errors in config.
- **Does not**: read files, call APIs, or parse command input.

### `internal/files/read.go`
- **Does**: expand globs (`filepath.Glob` plus `filepath.Walk` for `**`), read file contents, return per-path strings with error reporting.
- **Does not**: call APIs, mutate disk state, or parse JSON/JSONL.

### `templates/delegation-rules.template.md`
- **Does**: contain vendor-neutral copy-paste instructions for *any* LLM coding agent (Claude Code, Cursor, Copilot Chat, Aider, etc.) telling it when to call `sancho ask` / `sancho write`. Uses the tool name `sancho`, not a vendor-specific name.
- **Does not**: mention Claude-specific magic files, bash, or non-portable instructions.

## 5) Provider abstraction

All providers implement the `Provider` interface defined in `provider.go`:

```go
type Provider interface {
    ChatCompletion(ctx context.Context, req ChatRequest) (*ChatResponse, error)
    SupportsModel(model string) bool
}

type ChatRequest struct {
    Model    string    `json:"model"`
    Messages []Message `json:"messages"`
    MaxTokens int     `json:"max_tokens,omitempty"`
}

type ChatResponse struct {
    Content     string
    Usage       Usage
    FinishReason string
}

type Usage struct {
    PromptTokens     int
    CompletionTokens int
    TotalTokens    int
}
```

### Retry policy (`internal/client/retry.go`)
- **Does**: wraps the provider's `ChatCompletion` with automatic retries for transient HTTP 429/502/503/504, exponential back-off capped at 5 s, max 3 attempts (configurable via `retry` block in config file).
- **Does not**: retry 400/401/403 (client error), or infinite-loop on persistent failure. Implements as middleware that accepts any `Provider` and returns a wrapped `Provider`.

### Timeouts (`internal/client/timeout.go`)
- **Does**: wraps any `Provider` with per-request deadlines via `context.WithTimeout` (default 120 s from `timeout_seconds` config or `--timeout` flag).
- **Does not**: rely on `http.Client.Timeout` alone — uses context for proper cancellation.

### Security
- **Does**: API key is read once, held in memory as `string`, never logged or written to stderr/stdout.
- **Does not**: print headers, request bodies, or full response JSON to stderr (only usage summary).
- Config file permissions: warn (stderr) if `.sancho.json` is world-readable (mode & os.FileMode(0022) != 0), but do not abort.

### Stability / self-healing
- **Does**: if provider returns retryable error with `retry_after`, honour it when present.
- **Does**: treat empty response content as a failure (retry once, then surface clear stderr error with suggested `--max-tokens` increase).

## 6) Config file loader

**Search order** (first match wins):
1. `.sancho.json` in CWD
2. `$HOME/.config/sancho/config.json`

**Loader contract**:
- Read file
- Run regex pass: strip `//.*$` and `/\*.*?\*/` (non-greedy, dotAll)
- `json.Unmarshal` into a `Config` struct
- Return merged config; missing fields stay zero-valued so env fallback still applies.

**`Config` struct** (mirrors JSON schema above):

```go
type Config struct {
    APIKey  string `json:"api_key"`
    BaseURL string `json:"base_url"`
    Model   string `json:"model"`
    Provider string `json:"provider"` // "openrouter", "bedrock", "openai", "anthropic"

    OpenRouter OpenRouterConfig `json:"openrouter"`
    Bedrock    BedrockConfig    `json:"bedrock"`
    OpenAI     OpenAIConfig     `json:"openai"`
    Anthropic  AnthropicConfig  `json:"anthropic"`

    Ask    AskConfig    `json:"ask"`
    Write  WriteConfig  `json:"write"`
    Retry  RetryConfig  `json:"retry"`
    TimeoutSeconds int   `json:"timeout_seconds"`
}

type OpenRouterConfig struct {
    APIKey     string `json:"api_key"` // optional override
    HTTPReferer string `json:"http_referer"`
}

type BedrockConfig struct {
    Region        string            `json:"region"`
    ModelMappings map[string]string `json:"model_mappings"` // user-friendly name -> Bedrock ID
}

type OpenAIConfig struct {
    APIKey       string `json:"api_key"` // optional override
    Organization string `json:"organization"`
}

type AnthropicConfig struct {
    APIKey string `json:"api_key"` // optional override
}
```

## 7) Resolver

Merges the four layers into a final `Settings` value used by the rest of the app:

```
CLI flag → File → Env → Default
```

Example:
- `sancho ask --provider bedrock --model claude-3-opus` uses Bedrock with mapped model name.
- `SANCHO_PROVIDER` is used only when the file has no `provider` field.
- Fallback `WORKER_*` envs are consulted only when `SANCHO_*` is absent.

## 12) Provider implementations

Each provider implements the `Provider` interface. All share the same retry/timeout middleware.

### `internal/client/openrouter.go`
- Default provider. Uses API key from config, standard Bearer auth, required headers (`HTTP-Referer`, `X-Title`), POSTs to `/chat/completions`.
- Model format: `openrouter/*` or vendor-prefixed like `anthropic/claude-3-opus`.

### `internal/client/bedrock.go`
- Uses AWS SDK v2 credential chain, SigV4 signing.
- Model format: AWS model IDs (e.g., `anthropic.claude-3-opus-20240307-v1:0`).
- Auto-detects when `--model` starts with `bedrock/` prefix.

### `internal/client/openai.go`
- Uses `SANCHO_API_KEY` or `OPENAI_API_KEY` env var.
- Standard OpenAI chat completions endpoint.
- Model format: `gpt-4`, `gpt-3.5-turbo`, etc.

### `internal/client/anthropic.go`
- Uses `SANCHO_API_KEY` or `ANTHROPIC_API_KEY` env var.
- Uses `/v1/messages` endpoint with Claude-specific API format.
- Model format: `claude-3-opus-20240307`, etc.

## 9) Testing strategy

Every test file should be table-driven, deterministic, and free of live network calls.

### `cmd/ask_test.go`
- Tests that flag defaults are applied (max-tokens = 8192, provider = openrouter).
- Tests that `--model`, `--provider`, `--api-key`, `--base-url` override settings.
- Tests that `--json` flag suppresses cost footer.
- Uses `httptest.Server` to capture the request body and assert MinLang system prompt is present.

### `cmd/write_test.go`
- Tests that `--spec` and `--target` are required.
- Tests that `--context` is injected as `<reference>` block.
- Tests markdown fence stripping on model output before write.
- Tests human-readable stdout line contains target path and char count.

### `internal/client/provider_test.go`
- Tests `NewProvider` returns correct provider for each type.
- Tests default provider is "openrouter" when unspecified.
- Tests provider detection by model prefix (e.g., `bedrock/*`).

### `internal/client/openrouter_test.go`
- Tests `ChatCompletion` sets `Authorization`, `X-Title`, `HTTP-Referer`, `Content-Type`.
- Tests request body includes `model` and `messages`.
- Tests response parsing correctly extracts `content`, `prompt_tokens`, `completion_tokens`, `finish_reason`.

### `internal/client/bedrock_test.go`
- Tests SigV4 signing is applied to requests.
- Tests model name mapping to Bedrock format.
- Tests credential chain is used when no explicit API key.

### `internal/client/openai_test.go`
- Tests standard OpenAI endpoints are called.
- Tests model format passes through directly.

### `internal/client/anthropic_test.go`
- Tests `/v1/messages` endpoint is used.
- Tests Claude-specific request format (messages with cache_control, etc.).

### `internal/client/retry_test.go`
- Tests retry is triggered on 429/502/503/504, not on 400/401/403.
- Tests exponential back-off timing (mock time or short sleeps).
- Tests max attempts cap is respected.
- Tests `retry_after` from provider error body is honoured when present.
- Tests empty response content triggers one retry, then returns a clear error.

### `internal/client/timeout_test.go`
- Tests per-request timeout fires when server sleeps past deadline.
- Tests context cancellation short-circuits the request.

### `internal/config/env_test.go`
- Tests `SANCHO_API_KEY` is read, `WORKER_API_KEY` is fallback.
- Tests `SANCHO_BASE_URL`, `SANCHO_MODEL` fallback chain.
- Tests unset vars produce zero values (not panics).

### `internal/config/file_test.go`
- Tests `.sancho.json` is loaded over `$HOME/.config/sancho/config.json` when both exist.
- Tests JSONC comments are stripped before unmarshal.
- Tests missing file returns zero-valued config without error.

### `internal/config/resolve_test.go`
- Tests precedence: CLI flag > file > env > default.
- Tests partial configs (e.g. only `api_key` in file, model from env).
- Tests backwards-compat env vars are consulted when `SANCHO_*` absent.

### `internal/files/read_test.go`
- Tests single-level glob expansion (`*.go`).
- Tests `**` recursive walk via `filepath.Walk`.
- Tests non-existent path returns error.

## 11) Command UX / syntax constraints

Keep flags short, consistent, and memorable across all subcommands:
- `-p` / `--paths` for file lists (ask)
- `-q` / `--question` for the query (ask)
- `-s` / `--spec` for generation instructions (write)
- `-c` / `--context` for style reference (write)
- `-t` / `--target` for output path (write)
- `-m` / `--model` for model override (ask, write)
- `--provider` for provider override (ask, write)
- `-n` / `--max-tokens` for token budgets (ask, write)
- `--api-key` to override API key (ask, write)
- `--base-url` to override provider endpoint (ask, write)
- `--json` to suppress cost footer (ask)

No hidden flags, no `--config` flag in v1.
